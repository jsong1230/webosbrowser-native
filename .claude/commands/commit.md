## 1. 변경사항 확인
`git status`, `git diff`, `git diff --staged`

## 2. 논리적 단위로 분리
구현 코드 / 설계 문서 / 기술 문서 / 운영 문서 / 테스트 → 각각 별도 커밋

## 3. 커밋 메시지 (Conventional Commits + 한국어)
- `feat: 쿠폰 적용 API 구현`
- `docs: 쿠폰 기능 설계서 및 요구사항 분석서 작성`
- `docs: 쿠폰 API 스펙 및 DB 설계서 확정`
- `docs: 쿠폰 기능 진행 로그 및 CHANGELOG 업데이트`

## 4. 사용자 확인 후 실행

추가 지시사항: $ARGUMENTS
