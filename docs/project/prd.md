# webOS Browser — PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 프로젝트명
webOS Browser

### 목적
LG 프로젝터 HU175QW를 위한 리모컨 최적화 웹 브라우저를 개발하여, 가정용 엔터테인먼트 환경에서 대화면으로 편리하게 웹 콘텐츠를 즐길 수 있도록 합니다.

### 배경
- LG webOS 플랫폼은 스마트 TV/프로젝터용 운영체제로, 내장 브라우저가 제한적이거나 UX가 리모컨 조작에 최적화되지 않음
- 대화면 프로젝터 환경에서 웹 스트리밍(YouTube, Netflix), 웹 서핑, 온라인 미디어 소비 수요 증가
- 리모컨만으로 직관적이고 편리하게 조작 가능한 브라우저 필요

### 대상 사용자
- 가정용 프로젝터 사용자 (엔터테인먼트 목적)
- 리모컨만으로 웹 브라우저를 조작하고자 하는 일반 사용자
- 대화면 환경에서 영상 스트리밍, 뉴스, 블로그, 포털 사이트 이용

### 핵심 가치
- **리모컨 최적화**: 십자키, 방향키, 선택 버튼만으로 모든 기능 사용 가능
- **대화면 가독성**: 원거리에서도 잘 보이는 큰 폰트, 명확한 UI
- **빠른 접근**: 자주 사용하는 사이트를 북마크/홈페이지로 즉시 접근
- **안정성**: 주요 웹사이트(YouTube, Netflix, 포털)에서 안정적인 렌더링

## 2. 사용자 페르소나

### 페르소나 1: 가족 영화 시청자 (30대, 가정주부/직장인)
- **역할**: 주말 저녁 가족과 함께 프로젝터로 영화/드라마 시청
- **목표**: YouTube, Netflix, Disney+ 등 스트리밍 사이트를 쉽게 접근하여 콘텐츠 시청
- **불편점**:
  - 리모컨으로 URL 입력이 어렵고 느림
  - 북마크가 없어 매번 검색해야 함
  - 탭이 없어 여러 사이트를 번갈아 보기 힘듦
- **기대**:
  - 자주 가는 사이트를 북마크로 저장하고 빠르게 접근
  - 리모컨 단축키로 탭 전환, 뒤로/앞으로 이동

### 페르소나 2: 뉴스/정보 소비자 (40대, 직장인)
- **역할**: 출근 전 또는 저녁 시간에 뉴스, 블로그, 커뮤니티 사이트 확인
- **목표**: 포털 사이트, 뉴스 사이트를 리모컨으로 편리하게 탐색
- **불편점**:
  - 일반 브라우저는 마우스/터치패드 전제로 설계되어 리모컨 조작 어려움
  - 링크 클릭, 스크롤, 탭 전환이 불편
- **기대**:
  - 리모컨 십자키로 링크 간 포커스 이동
  - 히스토리 기능으로 이전에 본 페이지 빠르게 재접근

## 4. 비기능 요구사항

### 성능
- **페이지 로딩 속도**: 일반적인 웹 페이지 5초 이내 로딩 (webOS 제약 고려)
- **메모리 사용량**: 탭당 최대 200MB, 전체 브라우저 1GB 이하 (webOS 메모리 제약)
- **탭 전환 반응 속도**: 탭 전환 시 1초 이내 UI 전환
- **리모컨 입력 반응**: 버튼 입력 후 0.3초 이내 UI 반응

### UX (리모컨 최적화)
- **포커스 표시**: 현재 포커스된 요소(링크, 버튼)를 명확히 표시 (두꺼운 테두리, 하이라이트)
- **가독성**: 최소 폰트 크기 16px (3m 거리에서 가독 가능)
- **네비게이션**: 십자키 위/아래/좌/우로 모든 UI 요소 접근 가능
- **단축키**: 백 버튼, 홈 버튼 등 물리 버튼으로 주요 기능 빠른 실행
- **에러 복구**: 에러 발생 시 명확한 안내 + 재시도 옵션

### 보안
- **HTTPS 우선**: HTTPS 연결 우선, 비보안 사이트 경고
- **피싱 방지**: 주요 사이트(은행, 포털) 접속 시 URL 확인 안내 (선택)
- **쿠키/캐시 삭제**: 설정에서 브라우징 데이터 삭제 기능

### 호환성
- **webOS 버전**: webOS 4.x 이상 지원 (LG 프로젝터 HU175QW 기준)
- **주요 사이트 렌더링**: YouTube, Netflix, Naver, Google 등 정상 동작
- **웹 표준**: HTML5, CSS3, ES6+ 지원 (webOS Chromium 기반)

### 확장성
- **플러그인 구조**: 향후 광고 차단, 다크 모드 등 기능 추가 용이
- **설정 동기화**: 향후 LG 계정 연동으로 북마크/설정 동기화 가능 (선택)

## 5. 기술 제약조건

- **플랫폼**: webOS 기반 (LG TV/프로젝터 OS)
- **프레임워크**: webOS Enact (React 기반 UI 프레임워크)
- **웹뷰 API**: webOS에서 제공하는 iframe 또는 WebView API 사용 (제한적)
- **입력**: 리모컨 키보드 이벤트만 지원 (마우스, 터치 불가)
- **스토리지**: webOS 로컬 스토리지 (IndexedDB, LocalStorage)
- **네트워크**: webOS 네트워크 스택 사용 (CORS, 인증서 제약)

## 6. 범위 외 (Out of Scope)

### 이번 프로젝트에서 하지 않는 것
- **모바일/데스크톱 버전**: webOS 전용 (타 플랫폼 포팅 없음)
- **광고 차단**: 기본 기능에 포함하지 않음 (추후 확장 가능)
- **비디오 다운로드**: 스트리밍만 지원, 로컬 다운로드 불가
- **확장 프로그램**: Chrome 확장 기능 설치 불가
- **시크릿 모드**: 프라이버시 모드 미포함 (추후 확장 가능)
- **음성 검색**: 음성 입력 기능 미포함 (추후 확장 가능)
- **멀티 프로필**: 사용자별 프로필 분리 없음 (단일 사용자 가정)

## 7. 성공 지표

### 기능 완성도
- 핵심 기능(F-01~F-13) 100% 구현 및 정상 동작
- 주요 사이트(YouTube, Netflix, Naver, Google) 렌더링 성공률 95% 이상

### 성능
- 페이지 로딩 시간: 80% 이상의 페이지가 5초 이내 로딩
- 탭 전환 응답 시간: 1초 이내
- 앱 실행 시간: 3초 이내

### UX
- 리모컨 조작만으로 모든 핵심 기능 사용 가능 (마우스 불필요)
- 포커스 이동 경로가 직관적 (사용자 혼란 최소화)
- 에러 발생 시 복구 가능 (앱 크래시 없음)

### 보안
- HTTPS 사이트 정상 연결
- 비보안 사이트 접속 시 경고 표시

### 안정성
- 1시간 연속 사용 시 메모리 누수 없음
- 주요 사이트 10개 이상 탭 열어도 안정적 동작
