# 즐겨찾기 홈 화면 — 요구사항 분석서

## 1. 개요

### 기능명
즐겨찾기 홈 화면 (F-15)

### 목적
사용자가 자주 방문하는 웹사이트를 한눈에 볼 수 있는 시각적 홈 화면을 제공하여, 브라우저 시작 시 또는 홈 버튼 클릭 시 빠르게 원하는 사이트에 접근할 수 있도록 합니다.

### 대상 사용자
- 매일 동일한 웹사이트(YouTube, Netflix, 뉴스 사이트 등)를 방문하는 프로젝터 사용자
- 북마크 목록보다 시각적 그리드 뷰를 선호하는 사용자
- 리모컨으로 빠르게 자주 가는 사이트에 접근하고자 하는 일반 사용자

### 요청 배경
- 일반 데스크톱 브라우저(Chrome, Edge)는 새 탭 열기 시 자주 방문한 사이트를 타일 형태로 표시
- 프로젝터 대화면 환경에서 타일 그리드 UI는 원거리에서도 가독성이 높고 직관적
- 홈 버튼 클릭 시 빈 페이지 대신 자주 가는 사이트를 바로 보여주어 사용자 경험 향상
- 북마크(F-07)와 히스토리(F-08) 데이터를 활용하여 추가 데이터 입력 없이 자동 생성 가능

---

## 2. 기능 요구사항 (Functional Requirements)

### FR-1: 홈 화면 표시 (자주 가는 사이트 그리드)
- **설명**: 브라우저 시작 시 또는 홈 버튼 클릭 시 자주 방문한 사이트를 그리드 형태로 표시합니다.
- **유저 스토리**: As a 사용자, I want 브라우저를 시작하면 자주 가는 사이트가 한눈에 보이기를 원합니다, so that URL을 입력하지 않고 즉시 사이트에 접근할 수 있습니다.
- **우선순위**: Must
- **상세 요구사항**:
  - HomePage 컴포넌트 (홈 화면 전용 뷰)
  - 자주 가는 사이트 타일을 그리드 형태로 배치 (예: 3열 x 2행 = 6개 타일)
  - 각 타일 표시 항목:
    - **사이트 이름**: 북마크 제목 또는 페이지 제목
    - **파비콘/썸네일**: 사이트 아이콘 (없으면 기본 아이콘 표시)
    - **URL**: 작은 글씨로 URL 일부 표시 (선택적)
  - 타일 클릭 시 해당 사이트로 즉시 이동
  - 리모컨 방향키로 타일 간 포커스 이동 (좌/우/상/하)
  - 포커스된 타일은 하이라이트 표시 (두꺼운 테두리, 배경색 변화)

### FR-2: 자주 가는 사이트 자동 선정
- **설명**: 북마크와 히스토리 데이터를 기반으로 자주 방문한 상위 N개 사이트를 자동 선정합니다.
- **유저 스토리**: As a 사용자, I want 자주 방문한 사이트가 자동으로 홈 화면에 표시되기를 원합니다, so that 수동으로 관리하지 않아도 항상 최신 상태를 유지할 수 있습니다.
- **우선순위**: Must
- **상세 요구사항**:
  - **북마크 우선**: 북마크에 저장된 사이트 중 `visitCount`가 높은 순서로 우선 선정
  - **히스토리 보조**: 북마크가 N개 미만일 경우 히스토리에서 `visitCount`가 높은 사이트 추가
  - **표시 개수**: 6~9개 타일 표시 (설정 가능, 기본값: 6개)
  - **중복 제거**: 동일 도메인의 여러 URL은 대표 URL 하나만 표시 (예: `youtube.com/watch?v=...` → `youtube.com`)
  - **실시간 업데이트**: 북마크 추가/삭제, 페이지 방문 시 홈 화면 데이터 자동 갱신

### FR-3: 홈 버튼으로 홈 화면 이동
- **설명**: NavigationBar의 홈 버튼 또는 리모컨 홈 버튼 클릭 시 홈 화면으로 이동합니다.
- **유저 스토리**: As a 사용자, I want 홈 버튼을 클릭하면 홈 화면으로 돌아가기를 원합니다, so that 언제든지 자주 가는 사이트 목록으로 돌아갈 수 있습니다.
- **우선순위**: Must
- **상세 요구사항**:
  - NavigationBar에 홈 버튼 제공 (아이콘: 집 모양)
  - 홈 버튼 클릭 시:
    - 현재 탭의 URL을 홈 화면 URL로 변경 (예: `webos://home` 또는 `about:home`)
    - HomePage 컴포넌트 렌더링
  - 리모컨 홈 버튼(하드웨어 버튼)으로도 홈 화면 이동 가능 (선택적)
  - 홈 화면은 브라우저 히스토리에 포함되지 않음 (뒤로 가기로 홈 화면 재방문 방지)

### FR-4: 브라우저 시작 시 홈 화면 표시
- **설명**: 브라우저 앱 최초 실행 시 빈 페이지 대신 홈 화면을 표시합니다.
- **유저 스토리**: As a 사용자, I want 브라우저를 처음 열면 홈 화면이 나타나기를 원합니다, so that 즉시 자주 가는 사이트에 접근할 수 있습니다.
- **우선순위**: Must
- **상세 요구사항**:
  - 앱 최초 실행 시 첫 번째 탭의 초기 URL을 홈 화면으로 설정
  - 설정에서 홈페이지 URL 변경 가능 (예: 구글, 네이버 등 특정 사이트)
  - 사용자가 홈페이지 URL을 설정한 경우 홈 화면 대신 해당 URL로 이동

### FR-5: 타일 클릭 시 사이트 로드
- **설명**: 홈 화면에서 타일을 클릭하면 해당 사이트로 즉시 이동합니다.
- **유저 스토리**: As a 사용자, I want 타일을 클릭하면 해당 사이트가 즉시 열리기를 원합니다, so that 빠르게 웹 서핑을 시작할 수 있습니다.
- **우선순위**: Must
- **상세 요구사항**:
  - 타일 선택 버튼 클릭 시 해당 URL로 WebView 로드
  - 로딩 시작 시 로딩 인디케이터 표시 (F-05와 연계)
  - 로드 성공 시 홈 화면 숨김, WebView 콘텐츠 표시
  - 로드 실패 시 에러 메시지 표시 (F-10과 연계)
  - 새 탭에서 열기 옵션 제공 (컨텍스트 메뉴 또는 리모컨 추가 버튼)

### FR-6: 타일 편집 기능
- **설명**: 사용자가 홈 화면 타일을 수동으로 추가/삭제/순서 변경할 수 있습니다.
- **유저 스토리**: As a 사용자, I want 홈 화면 타일을 내가 원하는 대로 편집할 수 있기를 원합니다, so that 자주 가는 사이트를 직접 관리할 수 있습니다.
- **우선순위**: Should (선택적 기능)
- **상세 요구사항**:
  - 홈 화면 상단에 "편집" 버튼 제공
  - 편집 모드 진입 시:
    - 각 타일 우측 상단에 "X" 버튼 표시 (삭제용)
    - "+ 사이트 추가" 타일 표시 (새 타일 추가용)
  - 타일 삭제:
    - "X" 버튼 클릭 시 해당 타일 제거
    - 홈 화면 설정 데이터에 저장 (북마크/히스토리는 그대로 유지)
  - 타일 추가:
    - "+ 사이트 추가" 타일 클릭 시 북마크 목록에서 선택
    - 선택한 북마크를 홈 화면에 추가
  - 타일 순서 변경 (선택적):
    - 리모컨 방향키로 타일 이동 (좌/우 화살표로 순서 변경)
  - "완료" 버튼으로 편집 모드 종료

### FR-7: 홈 화면 데이터 영속성
- **설명**: 홈 화면 타일 구성을 로컬 저장소에 저장하여 앱 재시작 후에도 유지됩니다.
- **유저 스토리**: As a 사용자, I want 홈 화면 설정이 앱을 종료해도 유지되기를 원합니다, so that 다음 실행 시에도 동일한 홈 화면을 볼 수 있습니다.
- **우선순위**: Must
- **상세 요구사항**:
  - HomeScreenService 서비스: 홈 화면 타일 데이터 관리
  - 저장 데이터 구조:
    ```javascript
    {
      tileIds: ["bookmark-id-1", "bookmark-id-2", ...], // 타일 순서
      tileType: "auto" | "manual", // 자동/수동 모드
      tileCount: 6 // 표시할 타일 개수
    }
    ```
  - **자동 모드**: 북마크/히스토리 데이터 기반으로 자동 선정
  - **수동 모드**: 사용자가 편집한 타일 구성 유지
  - LocalStorage 또는 IndexedDB에 저장
  - 앱 초기화 시 저장된 설정 로드

### FR-8: 빈 홈 화면 처리
- **설명**: 북마크와 히스토리가 없을 경우 빈 홈 화면 안내 메시지를 표시합니다.
- **유저 스토리**: As a 신규 사용자, I want 홈 화면이 비어있으면 안내 메시지가 나타나기를 원합니다, so that 무엇을 해야 할지 알 수 있습니다.
- **우선순위**: Must
- **상세 요구사항**:
  - 북마크 0개, 히스토리 0개일 경우:
    - 안내 메시지 표시: "자주 가는 사이트가 없습니다. 북마크를 추가하거나 웹 사이트를 방문해보세요."
    - "URL 입력" 버튼 제공 (URLBar로 포커스 이동)
    - "북마크 관리" 버튼 제공 (BookmarkPanel 열기)
  - 기본 추천 사이트 제공 (선택적):
    - 빈 홈 화면 시 기본 사이트 타일 표시 (YouTube, Google, Naver 등)
    - 타일 클릭 시 해당 사이트 로드

### FR-9: 리모컨 키 매핑
- **설명**: webOS 리모컨 물리 버튼으로 홈 화면을 조작합니다.
- **유저 스토리**: As a 리모컨 사용자, I want 리모컨 버튼만으로 홈 화면을 탐색하고 사이트를 열 수 있기를 원합니다, so that 마우스 없이 편리하게 사용할 수 있습니다.
- **우선순위**: Must
- **상세 요구사항**:
  - **방향키 상/하/좌/우**: 타일 간 포커스 이동 (그리드 네비게이션)
  - **선택 버튼**: 포커스된 타일의 사이트 로드
  - **홈 버튼**: 홈 화면으로 이동 (이미 홈 화면이면 동작 없음)
  - **백 버튼**: 이전 페이지로 이동 (홈 화면에서는 동작 없음)
  - **컬러 버튼 (선택적)**:
    - 빨강: 타일 편집 모드 진입
    - 초록: 현재 페이지 북마크 추가

## 3. 비기능 요구사항 (Non-Functional Requirements)

### 성능
- **홈 화면 로딩 시간**: 앱 시작 시 홈 화면 데이터 로드 및 렌더링 1초 이내
- **타일 클릭 반응 속도**: 타일 클릭 후 페이지 로드 시작까지 0.3초 이내
- **북마크/히스토리 조회 성능**: 북마크 100개, 히스토리 1,000개 이상일 때도 홈 화면 생성 1초 이내
- **메모리 사용량**: 홈 화면 컴포넌트는 최대 50MB 이하

### 가독성 (대화면 최적화)
- **타일 크기**: 각 타일 최소 200x200px (3m 거리에서 가독 가능)
- **폰트 크기**:
  - 사이트 이름: 최소 24px
  - URL 서브 텍스트: 최소 16px
- **대비**: 배경과 전경 색상 대비 비율 4.5:1 이상 (WCAG AA 기준)
- **포커스 표시**: 포커스된 타일은 3px 두께 테두리 또는 명확한 배경색 변화
- **아이콘 크기**: 파비콘 최소 64x64px (원거리 가독성)

### UX (리모컨 최적화)
- **그리드 네비게이션**: 리모컨 방향키로 타일 간 포커스 이동이 직관적이고 자연스러움
- **포커스 초기 위치**: 홈 화면 진입 시 첫 번째 타일에 포커스 자동 설정
- **포커스 루프**: 마지막 타일에서 방향키 우측/하단 입력 시 첫 번째 타일로 순환 (선택적)
- **애니메이션**: 타일 클릭 시 부드러운 페이드 아웃 애니메이션
- **에러 복구**: 타일 로드 실패 시 에러 메시지 + "재시도" 버튼 제공

### 접근성
- **스크린 리더 지원**: 각 타일에 `aria-label` 속성 추가 (예: "YouTube로 이동")
- **키보드 네비게이션**: 방향키 외에도 Tab 키로 타일 간 이동 가능 (webOS 호환성 확인 필요)
- **고대비 모드**: 다크/라이트 테마에서 모두 명확한 타일 구분

### 확장성
- **타일 개수 조정**: 설정에서 타일 개수 변경 가능 (6개, 9개, 12개 등)
- **타일 레이아웃**: 그리드 레이아웃 (3x2, 4x2 등) 설정 가능
- **커스텀 썸네일**: 향후 사이트 스크린샷 캐싱으로 파비콘 대신 썸네일 표시 가능
- **홈 화면 테마**: 배경 이미지, 색상 테마 커스터마이징 가능 (추후 확장)

## 4. 제약조건

### webOS 플랫폼 제약사항
- **리모컨 전용 입력**: 마우스, 터치, 물리 키보드 사용 불가
- **Spotlight 시스템**: Enact의 Spotlight 시스템을 통해 타일 간 포커스 관리 필수 (Spotlightable 컴포넌트 사용)
- **5-way 방향키**: 상/하/좌/우/선택 버튼만으로 모든 입력 처리
- **화면 해상도**: 4K UHD (3840x2160) 기본, Full HD (1920x1080) 호환

### 기술 제약사항
- **프레임워크**: Enact 프레임워크 기반 (React 컴포넌트)
- **UI 컴포넌트**: Enact Moonstone UI 컴포넌트 우선 사용 (GridListImageItem, ImageItem 등)
- **CSS Modules**: 스타일링은 `.module.less` 파일로 분리
- **상태 관리**: React Hooks (useState, useEffect, useContext)
- **데이터 저장**: LocalStorage (홈 화면 설정), IndexedDB (북마크/히스토리 데이터)
- **파비콘 제약**: webOS WebView에서 파비콘 추출 제한 가능 (기본 아이콘 대체 필요)

### 의존성
- **F-07 (북마크 관리)**: 북마크 데이터 및 `visitCount` 필요 (완료)
- **F-08 (히스토리 관리)**: 히스토리 데이터 및 `visitCount` 필요 (완료)
- **F-02 (웹뷰 통합)**: 타일 클릭 시 WebView로 페이지 로드 필요 (완료)
- **F-04 (페이지 탐색 컨트롤)**: NavigationBar에 홈 버튼 추가 필요 (완료)
- **F-05 (로딩 인디케이터)**: 타일 클릭 후 로딩 중 인디케이터 표시 (완료)

### 범위 외 (Out of Scope)
- **사이트 스크린샷 캐싱**: 파비콘 대신 사이트 스크린샷 표시 미지원 (추후 확장 가능)
- **홈 화면 위젯**: 날씨, 시계 등 위젯 기능 미지원
- **다중 홈 화면**: 여러 페이지의 홈 화면 미지원 (1개 홈 화면만 제공)
- **타일 드래그 앤 드롭**: 리모컨 환경에서 드래그 앤 드롭 미지원 (순서 변경은 방향키로만)
- **홈 화면 공유**: 다른 사용자와 홈 화면 설정 공유 미지원

## 5. 인터페이스 요구사항

### 컴포넌트 간 연동

#### HomePage ↔ bookmarkService
- `getAllBookmarks()`: 모든 북마크 조회
- `incrementVisitCount(id)`: 타일 클릭 시 북마크 방문 횟수 증가
- 북마크 데이터 구조: `{ id, title, url, visitCount, favicon }`

#### HomePage ↔ historyService
- `getAllHistory()`: 모든 히스토리 조회
- 히스토리 데이터 구조: `{ id, title, url, visitCount, visitedAt, favicon }`

#### HomePage ↔ WebView (App.js)
- `onTileClick(url)`: 타일 클릭 시 WebView로 URL 전달
- WebView가 해당 URL 로드 시작

#### HomePage ↔ NavigationBar
- NavigationBar의 홈 버튼 클릭 이벤트 수신
- 홈 버튼 클릭 시 HomePage 컴포넌트 렌더링

#### HomePage ↔ HomeScreenService (신규)
- `getHomeScreenConfig()`: 홈 화면 설정 로드 (타일 순서, 개수 등)
- `saveHomeScreenConfig(config)`: 홈 화면 설정 저장
- `getTopSites(count)`: 북마크/히스토리 기반 상위 N개 사이트 조회

### 데이터 흐름
```
앱 시작
  → HomePage 컴포넌트 렌더링
    → HomeScreenService.getTopSites(6)
      → bookmarkService.getAllBookmarks() (visitCount 기준 정렬)
      → historyService.getAllHistory() (visitCount 기준 정렬)
      → 상위 6개 사이트 반환
    → HomePage에 타일 그리드 표시
  → 사용자가 타일 클릭
    → onTileClick(url) 호출
    → WebView로 URL 로드
    → bookmarkService.incrementVisitCount(id) (북마크인 경우)
```

## 6. 용어 정의

| 용어 | 정의 |
|------|------|
| **홈 화면** | 브라우저 시작 시 또는 홈 버튼 클릭 시 표시되는 자주 가는 사이트 그리드 뷰 |
| **타일** | 홈 화면에 표시되는 각 웹사이트의 아이콘 + 제목 카드 |
| **HomePage** | 홈 화면을 렌더링하는 React 컴포넌트 |
| **HomeScreenService** | 홈 화면 타일 데이터 관리 서비스 (상위 사이트 조회, 설정 저장) |
| **자주 가는 사이트** | 북마크 또는 히스토리에서 `visitCount`가 높은 상위 N개 사이트 |
| **그리드 네비게이션** | 리모컨 방향키로 타일 간 포커스를 이동하는 UI 패턴 (상/하/좌/우) |
| **자동 모드** | 북마크/히스토리 데이터를 기반으로 홈 화면 타일을 자동 생성하는 모드 |
| **수동 모드** | 사용자가 타일을 직접 추가/삭제/순서 변경한 홈 화면 구성 |
| **파비콘** | 웹사이트의 작은 아이콘 이미지 (16x16px 또는 32x32px) |
| **visitCount** | 북마크 또는 히스토리에서 해당 사이트를 방문한 횟수 |

## 7. 완료 기준 (Acceptance Criteria)

### AC-1: 홈 화면 표시
- [ ] 브라우저 시작 시 홈 화면이 첫 번째 탭에 표시됨
- [ ] 홈 화면에 자주 가는 사이트 타일이 그리드 형태로 표시됨 (기본 6개)
- [ ] 각 타일에 사이트 이름, 파비콘 표시
- [ ] 리모컨 방향키로 타일 간 포커스 이동 가능
- [ ] 포커스된 타일은 명확하게 하이라이트 표시

### AC-2: 자주 가는 사이트 자동 선정
- [ ] 북마크에서 `visitCount` 높은 순서로 사이트 선정
- [ ] 북마크가 6개 미만이면 히스토리에서 추가 선정
- [ ] 동일 도메인 중복 제거 (대표 URL만 표시)
- [ ] 북마크 추가/삭제 또는 페이지 방문 시 홈 화면 데이터 자동 갱신

### AC-3: 홈 버튼으로 홈 화면 이동
- [ ] NavigationBar에 홈 버튼 표시
- [ ] 홈 버튼 클릭 시 현재 탭에서 홈 화면으로 이동
- [ ] 홈 화면은 브라우저 히스토리에 포함되지 않음

### AC-4: 타일 클릭 시 사이트 로드
- [ ] 타일 선택 버튼 클릭 시 해당 URL로 WebView 로드
- [ ] 로드 시작 시 로딩 인디케이터 표시
- [ ] 로드 성공 시 WebView 콘텐츠 표시
- [ ] 로드 실패 시 에러 메시지 표시

### AC-5: 타일 편집 기능 (선택적)
- [ ] 홈 화면 상단에 "편집" 버튼 표시
- [ ] 편집 모드 진입 시 각 타일에 "X" 버튼 표시
- [ ] "X" 버튼 클릭 시 타일 삭제
- [ ] "+ 사이트 추가" 타일 클릭 시 북마크 선택 다이얼로그 표시
- [ ] 선택한 북마크를 홈 화면에 추가
- [ ] "완료" 버튼으로 편집 모드 종료

### AC-6: 홈 화면 데이터 영속성
- [ ] 홈 화면 타일 구성을 LocalStorage에 저장
- [ ] 앱 재시작 후 저장된 설정 정상 로드
- [ ] 자동 모드와 수동 모드 구분 저장

### AC-7: 빈 홈 화면 처리
- [ ] 북마크 0개, 히스토리 0개일 경우 안내 메시지 표시
- [ ] "URL 입력" 버튼 클릭 시 URLBar로 포커스 이동
- [ ] "북마크 관리" 버튼 클릭 시 BookmarkPanel 열기

### AC-8: 리모컨 키 매핑
- [ ] 방향키 상/하/좌/우로 타일 간 포커스 이동
- [ ] 선택 버튼으로 타일의 사이트 로드
- [ ] 홈 버튼으로 홈 화면 이동
- [ ] 모든 기능을 마우스 없이 리모컨만으로 사용 가능

### AC-9: 성능 및 가독성
- [ ] 홈 화면 로딩 시간 1초 이내
- [ ] 타일 클릭 반응 속도 0.3초 이내
- [ ] 북마크 100개, 히스토리 1,000개 이상일 때도 홈 화면 생성 1초 이내
- [ ] 타일 크기 최소 200x200px
- [ ] 사이트 이름 폰트 크기 최소 24px
- [ ] 포커스된 타일이 명확하게 구분됨

### AC-10: 회귀 테스트
- [ ] 브라우저 시작 → 홈 화면 표시 → 타일 클릭 → 사이트 로드 전체 흐름 정상 동작
- [ ] 홈 버튼 클릭 → 홈 화면 이동 → 타일 클릭 → 사이트 로드 정상 동작
- [ ] 북마크 추가 → 홈 화면 갱신 → 새 북마크 타일 표시
- [ ] 타일 편집 모드 → 타일 삭제 → 홈 화면 갱신
- [ ] 앱 재시작 → 홈 화면 설정 유지

### 기술적 고려사항
- **visitCount 계산**: 북마크와 히스토리의 `visitCount`는 별도로 관리되므로, 홈 화면 생성 시 두 데이터를 병합하여 총 방문 횟수 계산 필요
- **도메인 중복 제거**: `youtube.com/watch?v=abc`와 `youtube.com/watch?v=xyz`는 동일 도메인으로 간주하여 하나만 표시
- **파비콘 캐싱**: IndexedDB에 파비콘 URL 저장, 없으면 기본 아이콘 표시
- **홈 화면 갱신 타이밍**: 북마크 추가/삭제, 페이지 방문 시 홈 화면 데이터 자동 갱신 (실시간 또는 홈 화면 진입 시)
- **성능 최적화**: 북마크/히스토리 조회 후 메모리 캐싱으로 반복 조회 방지

### 병렬 작업 가능성
- F-15는 F-07, F-08 완료 후 단독 개발 권장 (의존성 높음)
- HomePage 컴포넌트와 HomeScreenService는 순차 개발 권장 (서비스 먼저 구현 후 컴포넌트)
